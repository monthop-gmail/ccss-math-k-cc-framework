# Workflow (TPT)

เป้าหมายของ workflow นี้คือทำให้ “ทำซ้ำเร็ว” และออกสินค้าได้สม่ำเสมอ โดยยังคุมคุณภาพ/ความถูกต้องตาม CCSS

## 0) หลักการ

- 1 product/ชุด = โฟกัส 1 standard (เช่น `K.CC.B.5`)
- เริ่มจาก template เดียว แล้วแตกธีม/ระดับความยาก (ง่าย -> ยาก)
- ทำ bundle ทุก 6–8 products

## 1) Production Loop (ต่อ 1 product)

1. เลือก `standard` + `sub-skill` + ช่วงตัวเลข (เช่น 0–10 หรือ 0–20)
2. เลือก `theme` (animals/food/space/seasonal)
3. Copy โครงจาก `templates/product-pack/` แล้วเริ่มเขียน `src/CONTENT.md`
4. สร้างชุดหน้า (page set) ตามโครงเดียวกัน
5. ทำ answer key + ตรวจ math accuracy
6. Export (PDF) + thumbnail + listing copy
7. อัปโหลดขึ้น TPT และทำ keyword/SEO ให้ตรงรหัสมาตรฐาน

## 2) Definition of Done

- มี cover/page instruction + practice + review
- มี answer key ทุกหน้า
- ชื่อไฟล์/ชื่อสินค้า มีรหัสมาตรฐาน (เช่น `K.CC.B.5`) ชัด
- Listing มีคำหลัก: standard + keyword หลัก + ช่วงตัวเลข + theme

## 3) Naming Convention (แนะนำ)

- Folder: `examples/k-cc-b5-how-many-0-10-animals/`
- Product title: `K.CC.B.5 How Many? Counting 0-10 Worksheets (Animals)`
- Export: `export/K.CC.B.5-how-many-0-10-animals.pdf`
